1. Architectural views

  1. Logical views

Logical views represent the logical decisions that were made in development, and how those decisions are to be implemented within the system.

  1. Decomposition View

The Decomposition View represents a breakdown of the functional pieces of the system as well as shows which functional modules belong to which subsystems in an is-part-of relationship. The view can be extended to include more or less detail within the modules, and is useful for maintenance and future development.

        

        

  1. Decomposition Element Catalog
  • Trade Net Front End App:
  • User-facing GUI
  • Sends requests and displays results
  • Account Management:
  • All legacy Account Management and Banking software
  • Trade Execution:
  • Tradier API interface
  • Execute transactions

3.1.3 Data Model View

The Data Model view represents the logical structure of the information and data being stored and used by the system. It is useful to developers and as well as any maintainers of the system.

3.1.3.1 Data Model Element Catalog

        User Account

  • Contains the financial information of user, user’s portfolio, and data of profits and losses.

        Transaction History

  • Records transaction made by user account.

        Portfolio

  • Contains information about user’s owned stock, the stock’s current price, the price of the stock when purchased, current profit and loss of the stocks in portfolio.

        Stock

  • Store information about the individual stock.

        Customer Data Report

  • Contains information about monthly profits and losses.

  1. Run-time Structure and Dynamic Views

The Run-time Structure and Dynamic Views show how the system will function after deployment and while in use by Users. This includes how the modules interact as well the signals and requests generated between modules and components.

  1. Tiered Client-Server Layered View

The Tiered Client-Server architectural view shows the interaction of specific elements and the layout of the system at run time. This allows for easier maintenance as well extendability for future functionality.

        

  1. Tiered Client-Server Element Catalog
  • Client Tier:
  • The tier containing all the different portals or methods for the user to access and use the system.
  • Web Tier:
  • Tier containing the validation checking for the session variables that would be instantiated if the user has logged in. Also includes the functional PHP pages that make calls to the database and to the Tradier API.
  • Back-end:
  • Tier containing the applications and databases stored on the server.

  1. Service-Oriented View

        The Service-Oriented View represents the collection of distributed components that provide and/or consume services and resources within the context of the system.

        

  1. Service-Oriented Element Catalog

Web browser

  • User web browser of choice (ie Chrome, Firefox)

Trade Net App

  • System that gives users real time stock information, access to portfolio, access to and creation of brokerage account, buy and sell stocks.

Trade Net Database

  • Contains user information (includes brokerage accounts, portfolio, company data access level, etc)

Legacy Management Application

  • Contains user banking information (non-brokerage accounts, balance, etc)

Trade Net Server

  • Communicate with Tradier for Market Data, Execute Trades, access user information.

Tradier API

  • Provides market data.

Dow Jones News

  • Provides real time news updates.

Stock Trade

  • Provides information about AMEX, NYSE, and NASDAQ.

  1. Pipe and Filter View

        The Pipe and Filter view provides the overview of the passage of data from the user when buying and selling shares of stock in run time as well as when users request company information. The pipes are buffered in case multiple users are buying and/or selling shares. The blocking behavior give priority to those that are selling stock. In the case of requesting company information each request is done concurrently with buffering when authenticating the user access.

  1. Allocation View

Allocation views are used to analyze performance, reliability, and the security of a system by showing how components are connected within the system and how the components are stored on the system.

  1. Deployment View

The Deployment View shows the system server components, the modules that are installed on each server, and how these modules communicate. This is useful for debugging in development and maintenance, and analysis of the system to verify requirements.
