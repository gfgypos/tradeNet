Consistency among Architectural Views (Mapping Between Views)

To appropriately and accurately describe the Trade Net Brokerage System to stakeholders, multiple architectural views were designed to describe the system from varying perspectives. The goal of each view is to address a specific concern from a stakeholder, however, all views must describe the system consistently. To properly map between views, each architectural view was derived from a standard of requirements and referenced the decomposition view; the benefit of creating the decomposition view first is that it maps out functionality, so creating views consistent with functionality and the requirements should net a consistent representation of all views of the system.
